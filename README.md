# Social inverse planning

Code and anonymized data accompanying the CogSci 2026 paper:

> Chen, A. M., Thomas, A. J., Tenenbaum, J. B., & Saxe, R. (2026). A social inverse-planning model captures how relationship intimacy constrains action planning and interpretation. *Proceedings of the Annual Conference of the Cognitive Science Society*.

The paper extends standard inverse-planning models so that the utilities of actions depend not only on physical properties (effort, outcome) but also on the structure of the relationship between the actors. Across three experiments using food sharing as a case study, we test how human observers reason about actions, desires, and relationships given both physical and sociological constraints.

The three experiments included here are:

- `food_forw_intimacy_desire` — Experiment 1, forward planning (predicting joint actions given desire and intimacy)
- `food_inv_intimacy_desire_alt` — Experiment 2a, intimacy inference (inferring intimacy from an observed action)
- `food_inv_desire_intimacy_alt` — Experiment 2b, desire inference (inferring desire from an observed action)

## Repository layout

```
cogsci-cr/
├── data/                       # anonymized participant data, with codebook in data/README.md
├── experiments/                # jsPsych code for the three experiments
├── model/
│   ├── stipulated_tables.py        # action-only V, access, and effort tables
│   ├── tables.py                   # scenario labels, enums, and table accessors
│   ├── utility.py                  # utility functions for the three model variants
│   ├── actors.py                   # forward-planning actor models
│   ├── observers.py                # Bayesian observer models for inverse planning
│   ├── forward/, inverse/, cv/     # per-experiment fit and cross-validation scripts
│   ├── outputs/                    # fit results, predictions, and CV folds (regenerated by make)
│   └── test_model_compliance.py    # sanity tests for table shapes, ablations, and predictions
├── analysis/                   # Quarto (R) notebooks that render the paper figures
├── scripts/                    # auxiliary scripts (e.g. regenerating paper macros)
├── figures/                    # figures rendered by the analysis notebooks
├── Makefile                    # end-to-end pipeline; run `make help` for the target list
├── pyproject.toml, uv.lock     # Python deps, managed with uv
└── renv.lock                   # R deps, managed with renv
```

## Setup

For Python we use [uv](https://github.com/astral-sh/uv) and for R we use [renv](https://rstudio.github.io/renv/) to manage packages.

```bash
uv sync                          # creates .venv and installs Python deps
R -e 'renv::restore()'           # installs the pinned R packages
```

## Reproducing the results

```bash
make all
```

This runs (in order) `make fits` → `make cv` → `make values` → `make figures`. Each target can also be run on its own; see `make help` for the full list. The model compliance tests can be run with `make test`.

The four analysis notebooks in `analysis/` are rendered by `make figures` and write their plots to `figures/`. 

## Data

Anonymized participant data for all three experiments is in `data/`, with one subdirectory per experiment. See [`data/README.md`](data/README.md) for the codebook.
