# cogsci-cr — Camera-ready CogSci 2026 subset

Self-contained code, data, and analysis for the camera-ready version of the
CogSci 2026 paper. The model uses the journal-version utility shape

```
U(a | s, I, m) = w_v · V(a, m) − w_d · access(a) · (1 − I)^γ − w_e · effort(a)
```

but with hand-stipulated, action-only values for V (desire), access (risk),
and effort (cost), rather than the LM-elicited per-scenario tables used in
the journal version. The values are broadcast across all 16 scenarios so the
existing actor and observer code shapes still work.

Stipulated values:

| Variable     | a=0 | a=1 | a=2 | a=3 |
|--------------|----:|----:|----:|----:|
| V_low        |   0 |  −1 |  −1 |  −1 |
| V_high       |   0 |   1 |   1 |   1 |
| access (risk)|   0 |   0 |   1 |   2 |
| effort (cost)|   0 |   1 |   1 |   1 |

Three experiments are covered:

- `food_forw_intimacy_desire` — Exp 1, forward planning
- `food_inv_intimacy_desire_alt` — Exp 2a, intimacy inference
- `food_inv_desire_intimacy_alt` — Exp 2b, desire inference

## Run

```bash
uv sync
uv run python model/forward/fit_food_forw_intimacy_desire.py
uv run python model/forward/predict_food_forw_intimacy_desire.py
uv run python model/inverse/fit_food_inv_intimacy_desire_alt.py
uv run python model/inverse/fit_food_inv_desire_intimacy_alt.py
uv run python model/inverse/predict_food_inv_intimacy_desire_alt.py
uv run python model/inverse/predict_food_inv_desire_intimacy_alt.py
uv run python model/cv/cv_food_forw_intimacy_desire.py
uv run python model/cv/cv_food_inv_intimacy_desire_alt.py
uv run python model/cv/cv_food_inv_desire_intimacy_alt.py

uv run python scripts/generate_values_tex.py  # regenerates cogsci-2026/values.tex

quarto render analysis/food-forw-intimacy-desire-analysis.qmd
quarto render analysis/food-inv-intimacy-desire-alt-analysis.qmd
quarto render analysis/food-inv-desire-intimacy-alt-analysis.qmd
quarto render analysis/inv-plan-combined-correlation.qmd
```

Both inverse fit scripts must run before either inverse predict, because the
predict step reads alpha_observer values from both fit_results files at once.

R packages are pinned in `renv.lock`. After opening `cogsci-cr.Rproj` in
RStudio, run `renv::restore()` once. The qmds use `here::here()`, which
anchors on `cogsci-cr.Rproj` so paths resolve relative to the project root.

Figures land in `figures/` (e.g. `food_forw_intimacy_desire_comparison.pdf`).

## Paper

The paper LaTeX source is in `cogsci-2026/`:

- `main.tex` — paper source
- `values.tex` — auto-generated macros for participant counts, fitted parameters, AIC, r, and α_obs values; regenerated by `scripts/generate_values_tex.py` (or `make values`) from the latest `data/` and `model/outputs/` content
- `cogsci.sty` — CogSci 2026 style
- `cogsci_bibliography_template.bib` — bibliography
- `figures/` — figures included in the paper (curated copies of plots from `figures/` at the repo root, plus hand-drawn schematic)
- `cogsci-2026-reviews.md` — reviewer comments on the conference submission

Do not edit `values.tex` by hand — changes are overwritten by the script. Update the source data or fits and re-run `make values` (or `make all`) to refresh.

`cogsci-2026/` is a nested git repository synced to Overleaf for collaborative editing. If you `git init` the cogsci-cr repo, add `cogsci-2026/` as a submodule, or remove `cogsci-2026/.git/` first if you want the LaTeX tracked directly.

## Notes on stipulation

The stipulated tables are action-only — broadcast across all 16 scenarios.
This means the model predicts the same actor probability for every scenario,
which is fine for the camera-ready paper but means the model has zero
across-scenario variance by construction. The analysis qmds were trimmed
accordingly: the "P(action) vs LM-derived risk" / "Belief updates vs
LM-derived risk" / "Residual vs risk" panels (which used per-scenario LM
risk on the x-axis) have been removed, since with stipulated values they
would collapse to a single vertical strip per action.
